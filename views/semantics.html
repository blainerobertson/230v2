HTML5 Semantics

HTML5 what?

Semantics! Yes, I realize this isn't a common word, but it means everything (by the way, that is a pun and it was intentional). In his article, Let's Talk about Semantics, Mike Robinson defines semantics as "Semantics are the implied meaning of a subject, like a word or sentence."

When we refer to HTML semantics we are talking about using the HTML element that describes the content that will be placed inside of it. For example, if you see <nav> you should be able to tell that inside of it will be navigation links. Likewise, if you see <ul> you can expect to find a list of items that are all equal and don't need to be in any particular order. That's semantics.

HTML5 introduced a large number of new elements, and each is meant to describe the type of content you would place inside of the element. Below are two lists of different elements, the first describes structural elements (those that add structure to a web page - think walls in a house to create rooms) and the second describes content elements (those that actually hold the content of the page).

No Semantic Meaning

If you have ever driven a standard transmission car you know that the clutch disengages the gears. It doesn't make the car go forward, nor does it make the car go backwards. It causes it to continue in the direction it is already headed. Thus, the clutch is a "non-gear", it has no meaning in terms of a direction.

You should note that the last item in the Structural Elements list is <div>, and the last element in the Content Elements list is <span>. These two elements have no semantic meaning! They are completely neutral. Why have them you may ask? The answer is that from time to time, we need an element that doesn't have any meaning in order to change its appearance or put around two or more other elements in order to imply proximity. They can be useful, but should be our element of last resort.

Automatically Accessible for Assistive Technology

What does that heading mean? When a site visitor requests your web page and they are using a screen reader or another similar device, using semantic structures actually tells the device where to look in the markup for particular types of content (e.g. the navigation bar or the main content). We don't have to add additional markup, just the semantic elements. This idea is more fully explained in this short article - https://www.paciellogroup.com/blog/2015/09/easy-content-organisation-with-html5/. I encourage you to take 10 minutes and read it.

Be Semantic

As you apply semantic elements to your web pages, it makes the pages easier for humans to read (as they look at your code) and easier for computer software (it knows where to look for the content it is searching for), for example all of the content of the page should be in the <main> element.
