Commit, Sync and Test

With the temporary home page complete it is ready for its initial public appearance.

  1. Save the index.html file. (There should not be a dot to the left of the file name in the brackets file list panel).
  2. Open the GitHub Desktop tool.
  3. The GitHub tool should show 1 file with uncommitted changes.
  4. Add a title to the commit title text box and a description of the new file.
  5. Click the "Commit" button below the description.
  6. Click the "Sync" button (top right corner) and wait for the animation to stop.
  7. Open a browser window and go to your remote repository site (e.g. userdemo22.github.io). The temporary home page should be visible.
  8. If the old page shows up in the browser window instead of the new one, try holding down your "Shift" key and click the "Reload" icon in the browser. (This tells the browser to ignore the stored (cached) version of the file and get a fresh version from the remote web server).
  9. If everything works “Hooray!!”
  10. If something fails, get help!

This exercise is complete only when you go to your remote repository using a web browser and see the temporary home page. Get help if needed but keep at it until everything works as it should.

Register your Site

With the temporary home page synced to your GitHub Repository you're ready to tell everyone where to find it. Do the following:

  1. In your browser, go to your Temporary home page on GitHub (the web version, not the code view).
  2. Click in the address bar, select the entire URL and copy it.
  3. Go to the Student Site list page in the Student Web Sites module of I-Learn.
  4. Follow the directions found on the page to create a link to your repository.
  5. When done, click your name and test the link to make sure it works.
  6. If it works, you're done. If not, rework it until it does.
