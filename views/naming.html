Web Naming Rules

Introduction

When working on the web there are many things that affect the operation of files - the browser, the protocol(s), the client and server operating systems, the language(s), etc.... While many of these are out of our control there are steps that we can take to help keep things consistent -- these steps are known as the web naming rules.

There are only four of them, but we should follow them religiously to avoid problems! These rules apply to all files and folders in the web site!

The Naming Rules

  1. Everything is lower case.
  2. No spaces are used.
  3. No special characters -- (e.g. <,>, \, /, #, ?, ! ).
  4. Use short, meaningful names.

Explanation

The case of file names is impacted by the server's operating system - Windows servers are case insensitive while Linux servers are case sensitive. To avoid problems with this, just make things lower case!

The Hyper-text Transfer Protocol (http) ignores spaces, except in file names. In file names it replaces a space with a symbol %20. This makes URL's look confusing and can also lend itself to confusion in the mind of site visitors. So, avoid using spaces. Instead, if you have to create a visual space, use under-scores _ or hyphens -.

Special characters often mean specific things to computers, so just avoid using them completely in names!

Short, meaningful names save you and site visitors from having to remember long complicated names for files and folders. They also, when meaningful, help with the predictive nature of the file or folder contents when working with those files or folders.

Summary

Be kind and helpful to yourself and site visitors by following these simple rules. Also, be aware that for the class you will be held accountable for doing so as this is part of the grading criteria!
