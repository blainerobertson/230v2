Greenville Weather Page

For the last several weeks you have been working with the Franklin weather page. This week you will add the Greenville weather page to the weather site.

Use the Weather.json data

As you did with the Franklin page, get the weather data from the weather.json file for the Greenville page. Use all weather-related data (not zip, longitude or latitude) for Greenville in the html page.

Valid and Semantic

Be sure to use valid and semantic (meaningful) html elements as you build the page and its data. The Greenville page should use the same layout as the Franklin page, but the weather data will be different.

Navigation

Activate the navigation links in both the Franklin and Greenville weather pages to work for both cities. The Springfield navigation item can remain non-operational. Remember to make the navigation a integral part of the UX.

Submission

  1. Upload the revised Franklin and new Greenville weather pages and CSS stylesheet used to style them to your web site.
  2. Do NOT create a new link to the Greenville page on your temporary home page.
  3. Both pages will be accessed using the Franklin weather page link that should already exist in your temporary home page.
  4. The new Greenville page will be accessed via the navigation bar in the Franklin page and vice versa.
  5. Be sure to test the operation of all links, pages and CSS on a variety of devices once the files are uploaded to your web site.
  6. It is strongly recommended that you exchange formative grading, using the grading matrix below, with at least one other classmate prior to the due date. This will allow you to catch errors and fix them prior to the actual assessment.

Grading Matrix

Objectives 1 and 2 are assessed in this week's work.

Objective 1 (20 points)

  1. The viewport meta element is present in the Greenville page - 2 points
  2. The Greenville page and all resources in the web site follow the web naming rules - 2 points
  3. The Greenville page has a meaningful, content related title - 2 points
  4. The Greenville page is constructed using semantic structural elements (including structural elements - header, nav, main, footer) and content elements - 4 points
  5. The hourly temperature data is included in the Greenville page - 2 points
  6. The Greenville page is valid HTML5 - 4 points
  7. The Greenville page layout matches that of the small and large wireframes on mobile and larger screen devices - 4 points (2 points per device)

Objective 2 (20 points)

  1. Flexbox is implemented in an external style sheet and is used to control the layout for mobile and large screen devices - 3 points (Yes or No)
  2. Flexbox changes occur within media queries for large screened devices - 3 points (Yes or No)
  3. All weather data (including hourly temps) is easily read, visually appropriate and can be accessed in all screen sizes - 3 points (Yes or No)
  4. The PARC and typography design principles are evident in the visual appearance of the Greenville page - 5 points (1 point will be deducted for each missing design principle)
  5. The CSS is valid CSS3 - 6 points (Minus 1 point for each CSS error up to 6 points)
