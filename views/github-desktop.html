GitHub Desktop Client

Introduction

To keep our interaction with GitHub manageable we will use the GitHub Desktop client (available for Mac and Windows). This activity will have you download, install and configure the tool to your computer. Part of the setup is to also create (clone) a local version of the web repository created in the first video.

When the desktop tool is setup, you will then build a simple web page in Brackets (or another web editor) and use the GitHub desktop tool to commit and sync the web page between your local and the remote repository and test that everything worked in a web browser.

Tool Setup and Repository Cloning

Follow the process described in the video below (pause and rewind as needed) to setup the tool and clone the remote web repository to your local machine.

Commit and Sync Content

In this video you will commit your content to the Git version controller and sync the content between the local and remote repositories. When done you will test that the web page you built can be seen in a browser. Remember to pause and rewind as needed until done. Check the outcomes list at the bottom of the page when finished.

Outcomes

  1. The GitHub Desktop Client was successfully downloaded and installed.
  2. My account username and password were successfully used to link the desktop client with my GitHub Account.
  3. The remote web repository was successfully cloned to my local computer.
  4. A simple web page was created, then committed and synced with my remote repository using the GitHub Desktop tool.
  5. The web page on the remote repository was successfully accessed using a web browser.

If any of these outcomes was not successful, go back and review the process and/or get help until all have been completed.
